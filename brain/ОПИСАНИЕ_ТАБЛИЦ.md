## 🗂️ Разделы и Таблицы

### 1️⃣ Базовая Инфраструктура (Core Infrastructure)

**Назначение:** Управление организациями, пользователями и командами

| Таблица | Описание | Ключевые поля |
|---------|----------|---------------|
| **tenants** | Организации (компании-клиенты платформы) | `clerkOrgId`, `name`, `slug`, `plan`, `status` |
| **users** | Пользователи системы (синхронизация с Clerk) | `clerkUserId`, `email`, `role`, `tenantId` |
| **teams** | Команды внутри организации | `name`, `tenantId` |
| **team_members** | Участники команд | `teamId`, `userId`, `role` |

**Связи:**
```
Tenant (1) ──→ (N) User
Team (1) ──→ (N) TeamMember ──→ (1) User
```

### 2️⃣ CRM - Управление Продажами (Sales CRM)

**Назначение:** Воронка продаж в стиле Pipedrive для страховых сделок

| Таблица | Описание | Ключевые поля |
|---------|----------|---------------|
| **pipelines** | Воронки продаж | `name`, `orderNr`, `active` |
| **stages** | Этапы воронки | `name`, `orderNr`, `probability` |
| **deals** | Сделки (потенциальные договоры) | `title`, `value`, `status`, `pipelineId`, `stageId` |
| **persons** | Контактные лица | `name`, `email`, `phone`, `orgId` |
| **organizations** | Компании-клиенты | `name`, `address`, `industry` |
| **insurance_products** | Страховые продукты | `name`, `category`, `baseRate` |
| **deal_coverages** | Покрытия в сделке | `dealId`, `productId`, `coverageAmount`, `premium` |

**Workflow:**
```
Pipeline → Stage → Deal → DealCoverage
                    ↓
              Person / Organization
```

**Статусы сделок:**
- `OPEN` - Активная сделка
- `WON` - Выиграна (конвертируется в Submission)
- `LOST` - Проиграна
- `DELETED` - Удалена

### 3️⃣ Андеррайтинг (Underwriting)

**Назначение:** Оценка рисков и выпуск полисов

| Таблица | Описание | Ключевые поля |
|---------|----------|---------------|
| **submissions** | Заявки на андеррайтинг | `company`, `amount`, `stage`, `riskLevel`, `underwriterId` |
| **policies** | Выпущенные полисы | `policyNumber`, `effectiveDate`, `expiryDate`, `grossPremium` |
| **claims** | Убытки/претензии | `claimNumber`, `policyId`, `amount`, `status` |

**Workflow:**
```
Deal (WON) → Submission → Policy → Claim
```

**Этапы Submission:**
1. `NEW` - Новая заявка
2. `REVIEW` - На проверке
3. `ANALYSIS` - Анализ рисков
4. `UNDERWRITING` - Андеррайтинг
5. `APPROVED` - Одобрена
6. `REJECTED` - Отклонена
7. `BOUND` - Связана (создан Policy)

**Уровни риска:**
- `LOW` - Низкий
- `MEDIUM` - Средний
- `HIGH` - Высокий
- `CRITICAL` - Критический

### 4️⃣ Финансовый Модуль (Financial Module)

**Назначение:** Выставление счетов, платежи, расчеты, мультивалютность

| Таблица | Описание | Ключевые поля |
|---------|----------|---------------|
| **invoices** | Счета | `invoiceNumber`, `totalAmount`, `status`, `dueDate` |
| **invoice_items** | Позиции счета | `invoiceId`, `title`, `quantity`, `rate`, `amount` |
| **payments** | Платежи | `paymentNumber`, `amount`, `currency`, `fxRate` |
| **scheduled_payments** | График платежей | `scheduledDate`, `amount`, `status`, `stage` |
| **credit_notes** | Кредит-ноты | `creditNoteNumber`, `amount`, `reason` |
| **payment_methods** | Способы оплаты | `name`, `type`, `active` |

**Workflow:**
```
Invoice → InvoiceItem
       → Payment
       → ScheduledPayment
       → CreditNote
```

**Особенности:**

**Рекуррентные счета:**
```prisma
recurring: true
repeatEvery: 30
repeatType: "DAYS"  // DAYS, WEEKS, MONTHS, YEARS
noOfCycles: 12
nextRecurringDate: "2026-03-06"
```

**Мультивалютность:**
```prisma
amount: 10000          // Сумма в EUR
currency: "EUR"
fxRate: 0.92          // Курс EUR/USD
amountInBaseCurrency: 10869.57  // В USD
bankCommission: 50
netAmount: 9950       // amount - bankCommission
```

**Статусы счетов:**
- `DRAFT` - Черновик
- `SENT` - Отправлен
- `VIEWED` - Просмотрен
- `PARTIALLY_PAID` - Частично оплачен
- `PAID` - Оплачен
- `OVERDUE` - Просрочен
- `CANCELLED` - Отменен
- `VOID` - Аннулирован

## 🎯 Основные Сценарии Использования

### Сценарий 1: Продажа → Полис

1. Создается **Deal** в CRM
2. Deal проходит по **Stages** в **Pipeline**
3. При статусе `WON` создается **Submission**
4. Андеррайтер проверяет **Submission**
5. При одобрении создается **Policy**
6. Выставляется **Invoice** с **InvoiceItems**
7. Клиент делает **Payment**
8. Все операции логируются в **AuditLogs**

### Сценарий 2: Убыток

1. По **Policy** поступает **Claim**
2. Прикрепляются **Documents**
3. Создаются **Activities** для обработки
4. Добавляются **Notes** и **Comments**
5. При одобрении выставляется **Invoice** (возврат)
6. Создается **Payment** или **CreditNote**

### Сценарий 3: Комплаенс

1. Создается **Organization** или **Person**
2. Запускается **ComplianceCheck**
3. Проверяются санкции, PEP, лицензии
4. Результат влияет на **Submission.riskLevel**
5. Все проверки документируются
